# 제주AI ON 협동조합 강의 예약 웹페이지 구현 설명서

본 문서는 **비개발자도 그대로 따라 할 수 있도록**  
제주AI ON 협동조합 강의 예약 웹페이지의 구현 과정을 단계별로 설명합니다.

---

## 1. 시스템 개요

### 1.1 구현 목표
- 제주AI ON 협동조합의 강의 프로그램을
  외부 신청자가 웹에서 간편하게 신청할 수 있도록 한다.
- 서버 구축 없이 **Google Sheets + Google Apps Script**를 활용한다.
- 운영자는 **코드 수정 없이 Google Sheets만 관리**한다.

---

### 1.2 전체 구조

[신청자]
↓ (웹 브라우저)
[강의 예약 웹페이지]
↓ (POST 요청)
[Google Apps Script 웹앱]
↓
[Google Sheets]


---

## 2. 파일 구성

| 파일명 | 역할 | 수정 여부 |
|------|------|---------|
| `index.html` | 강의 소개 + 신청 화면 | 문구 수정 시 |
| `style.css` | 디자인, 색상, 레이아웃 | 디자인 변경 시 |
| `script.js` | 강의 목록 조회, 신청 전송 | Apps Script URL 입력 |
| `apps-script.gs` | Sheets 연결 및 API | Google에 복사 |
| `구현설명서.md` | 운영/설치 설명서 | 문서 |

---

## 3. Google Sheets 설정

### 3.1 스프레드시트 생성

1. Google Drive 접속
2. `새로 만들기` → `Google 스프레드시트`
3. 파일명 변경:
제주AI_ON_강의신청목록

---

## 4. Google Apps Script 설정

### 4.1 Apps Script 열기

1. 스프레드시트 상단 메뉴 → `확장 프로그램`
2. `Apps Script` 클릭
3. 새 탭에서 편집기 열림

---

### 4.2 코드 붙여넣기

1. 기존 코드(`function myFunction(){}` 등) **전부 삭제**
2. 제공된 `apps-script.gs` 전체 코드 복사
3. Apps Script 편집기에 붙여넣기
4. 저장 (Ctrl + S)
5. 프로젝트 이름:
제주AI_ON_강의예약시스템

---

### 4.3 초기 세팅 실행 (필수)

1. 상단 함수 선택 드롭다운 → `setupSpreadsheet`
2. `실행` 클릭
3. 권한 요청 시:
- 권한 검토 → 본인 계정 선택
- “확인되지 않은 앱” → 고급 → 이동 → 허용
4. 스프레드시트에 아래 시트가 생성되면 성공:
- `Courses`
- `Reservations`

---

### 4.4 강의 목록 관리 (Courses 시트)

| 컬럼 | 설명 |
|----|----|
| courseId | 강의 고유 ID |
| title | 강의명 |
| summary | 한 줄 소개 |
| duration | 교육 시간 |
| target | 대상 |
| fee | 비용 |
| isOpen | TRUE = 노출 / FALSE = 숨김 |

📌 **강의 추가·수정은 이 시트만 관리하면 됨**

---

### 4.5 웹앱 배포

1. Apps Script 우측 상단 → `배포`
2. `새 배포` → 유형: `웹 앱`
3. 설정:
- 설명: `제주AI ON 강의 예약 v1`
- 실행 주체: 나
- 액세스 권한: 모든 사용자
4. `배포` 클릭
5. 생성된 **웹 앱 URL 복사**https://script.google.com/macros/s/XXXX/exec
